# SSOT 運用ガイド（長尺セッション向け）

> **目的**：進行中で長尺になりやすいチャットセッションを、
> 1つの Issue に“継続的に”集約・整流化するための最小ルール。

## いつ SSOT にするか
- セッションが**長尺**で、**数日にまたいで続く**と見込まれるとき
- 決定事項・合意の**履歴**を1スレッドで保ちたいとき

## SSOT の作り方（最小）
1. 新規 Issue を作成（タイトル例：`SSOT: <トピック>`）
2. **本文は骨子のみ**（目的／主要トピック／決定／未決、`[MASKED]`、**日付は書かない**）
3. 以降の更新は**コメントで差分追記**（中間 `— 続く`／最終 `— 完`／単独なし）
4. **Index へ 1 行 PREPEND**（失敗時はコメント代替→後日集約）

## コメント作法（差分主義）
- **既出は再掲しない**（参照リンクか要点のみ）
- **直近5件**に同趣旨がないか目視（重複抑止）
- 1コメント **≤30KB、≥10秒、1バッチ≤5**（失敗時は段落分割）
- 機微は **`[MASKED]`**

## checkpoint の使い所（任意）
- 方針・規約の変更
- 運用の節目（DRY-RUN/SAVE成功、週次運用の変更）
- 大きな節目の**チェックポイント要約**
- 構成変更（ディレクトリ/命名規則）

## 例：コメント末尾の表記
- 中間: `— 続く`
- 最終: `— 完`
- 単独: 末尾表記なし

## Index 運用（SSOTでも共通）
- **アンカー直下に PREPEND**（GFMの半角 `|`、ヘッダは「タイトル 1 列」）
- フォールバック（本文編集失敗時）はコメント代替→後日本文へ集約
